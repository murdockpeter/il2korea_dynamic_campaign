IL-2 Korea

Scenario Generator User Guide

This guide explains what each field in the current app actually changes in the generated mission, what gets exported into the game, and where the present limits still are.

Project: il2korea_dynamic_campaign
Audience: end users and testers

Overview

The generator does not build IL-2 Korea missions from zero. It clones a stock playable mission template and then patches the parts the app controls. That approach is deliberate because it is currently the most reliable way to produce missions that load and can be flown immediately.

Template Base
  • [DEMO]InchonStrike for UN/US-aligned player aircraft
  • [DEMO]BlackThursday for DPRK/PRC/Soviet-aligned player aircraft
What Gets Patched
  • Player aircraft
  • Friendly support aircraft
  • Briefing and mission text
  • Weather and time of day
  • Starting airfield
  • Light start-airfield ambiance
  • COOP packaging and server registration

UI Fields

Player Aircraft

What it changes

  • Sets the player-flyable aircraft type used by the player flight.
  • Sets the player coalition and therefore the default enemy coalition.
  • Limits which starting airfields are shown in the UI.
  • Influences which friendly support aircraft can be chosen.
  • Influences the parked-aircraft ambiance mix near the spawn.

Current flyable list

  • f51d
  • f80c10
  • f84e
  • f86a5
  • il10
  • la11
  • mig15bis
  • yak9p
This field does not yet choose an intelligent payload preset. It changes the aircraft type, not a full role-specific loadout package.
Target Type

What it changes

  • Filters the internal catalog to one target category.
  • Selects the briefing text profile used for the mission.
  • Sets the broad mission role bucket used for support logic.

Current target types

  • Airfield Strike
  • Bridge Strike
  • Ground Attack
  • Harbor Strike
  • Industrial Strike
  • Rail Interdiction
  • Troop Area Strike
The app chooses up to three matching catalog targets for text and scenario metadata, but it does not yet place a fully new target complex into a fresh procedural objective area.
Landscape

What it changes

  • HMap
  • Textures
  • Forests
  • GuiMap
  • SeasonPrefix
  • Date
  • Temperature
This changes mission environment settings, not a complete graph relocation to a brand-new part of the map.
Enemy Faction

Current behavior

In the current UI this field is automatically locked to the coalition opposite the selected player aircraft.

  • UN/US-aligned aircraft force a DPRK/PRC/Soviet-aligned enemy.
  • DPRK/PRC/Soviet-aligned aircraft force a UN/US-aligned enemy.

What it affects

  • Catalog target filtering by faction guess
  • Briefing text
Weather

What it changes

The app patches weather values directly into the mission options block.

  • Clear -> high ceiling, no precipitation, no fog
  • Broken Cloud -> medium clouds, light precipitation preset
  • Overcast -> lower ceiling, heavier cloud preset
  • Poor Visibility -> lower ceiling, more haze, layer fog enabled
Weather does not yet drive custom AI behavior, route selection, or mission difficulty scaling.
Start Time

What it changes

  • Patches the mission Time value directly.
  • Updates the generated briefing text with the takeoff time.

Current presets

  • 05:30 Dawn
  • 08:00 Morning
  • 12:00 Noon
  • 15:00 Afternoon
  • 18:30 Dusk

Default selection: 08:00.

Starting Airfield

What it changes

  • Moves the player start package to the selected field.
  • Moves the local apron ambiance package with the player package.
  • Updates briefing text.

Current curated choices

  • All aircraft: Auto
  • UN/US-aligned: Seoul (K-16), Kimpo (K-14), Yangsu-ri (K-49), Suwon (K-13)
  • DPRK/PRC/Soviet-aligned: Antung
Auto resolves to the stock template field. Blue defaults to Seoul. Red defaults to Antung.
COOP-Friendly Player Flight

What it changes

  • Enables multiplayer cooperative export mode.
  • Writes a multiplayer mission copy with MissionType = 1.
  • Marks the player flight for coop use.
  • Writes cooperative package files into the IL-2 Korea install.
  • Registers the mission in the local cooperative server setup JSON.

Extra outputs when enabled

  • data\Multiplayer\Cooperative\<MissionName>\
  • data\Multiplayer\COOP\
  • data\Multiplayer\
The preferred working path is the Great Battles-style cooperative package directory under data\Multiplayer\Cooperative\<MissionName>\.

Generation Logic

Friendly Support

Friendly non-player support flights are retargeted to aircraft appropriate to coalition and role.

  • Blue strike: f80c10, f84e, f86a5
  • Blue fighter: f51d, f80c10, f84e, f86a5
  • Red strike: la11, yak9p, mig15bis
  • Red fighter: la11, yak9p, mig15bis
Airfield Ambiance

The start area keeps a light stock-template atmosphere. The generator currently varies:

  • parked static aircraft near the start
  • a subset of non-critical support vehicles
  • a subset of non-critical parked aircraft

This is deliberately conservative so that the takeoff chain and mission parser stay stable.

Export Paths

Always Written
  • generated\
  • C:\Program Files\IL2Series\game\data\Missions\CodexGenerated
Additional COOP Outputs
  • C:\Program Files\IL2Series\game\data\Multiplayer\Cooperative\<MissionName>\
  • C:\Program Files\IL2Series\game\data\Multiplayer\COOP
  • C:\Program Files\IL2Series\game\data\Multiplayer

The generator also updates LocalServerSetup.json so the mission appears in Korea's cooperative host flow.

Current Limits

  • Objective geography is still template-based rather than fully procedural.
  • Mission logic is still stock-template derived.
  • Payload and loadout logic is still basic.
  • Blue and red starting airfields are curated, not exhaustive.
  • COOP export is working, but still intentionally conservative.
The project is presently optimized for one thing above all else: a generated mission that actually loads and can be flown in the current IL-2 Korea build.
This guide matches the current implementation in the app repository. If the generator logic changes, this file should be updated at the same time as README.md.
